// __contravariant(逆变) : 大类型(泛型父类类型) -> 小类型(泛型子类类型)，加上这个关键字向下转型无警告
@interface YSBag<__contravariant ObjectType> : NSObject
- (void)add:(ObjectType)object;
@end
YSBag<NSString *> *bag1 = nil;
YSBag<NSMutableString *> *bag2 = nil;
bag2 = bag1;
// __covariant(协变) : 小类型(泛型子类类型) ->
大类型(泛型父类类型)加上这个关键字不写强转代码也没有警告
@interface YSBag<__covariant ObjectType> : NSObject
- (void)add:(ObjectType)object;
@end
YSBag<NSString *> *bag1 = nil;

  • runtime简介

    • runtime简称运行时，也就是在运行时候的一些机制，其中最重要的就是消息机制.对于C语言，函数调用在编译时候就确定了，如果在编译阶段调用未实现的函数就会报错；而OC语言在编译时候并不能决定真正调用哪个函数，只有在真正运行的时候才会根据函数名称找到对应的函数来调用, 在编译阶段，OC可以调用任何函数，即使这个函数并未实现只要声明过就不会报错。
  • runtime作用
    • 发送消息
      • 方法调用本质，就是给对象发送消息
      • 用类名调用类方法，底层会自动把类名转换成类对象调用
      • 消息机制原理：对象根据方法编号SEL去映射表查找对应方法实现
    • 交换方法
Person *p = [[Person alloc] init];

    /* self:方法调用者 _cmd:调用方法的编号*/
void test(id self, SEL _cmd)
{
    NSLog(@"调用%@ %@",self, NSStringFromSelector(_cmd));
}
void drinkTest(id self,SEL _cmd, id param)
{
    NSLog(@"调用%@ %@ %@",self, NSStringFromSelector(_cmd), param);
}
+ (BOOL)resolveInstanceMethod:(SEL)sel
{
    if (sel == @selector(eat))
    {
        class_addMethod([self class], sel, (IMP) test, "v@:");
        return YES;
    }
    if(sel == @selector(drink:))
    {
        // @id对象 :SEL
        class_addMethod(self, sel, (IMP)drinkTest, "v@:@");
    }
    return [super resolveInstanceMethod:sel];
}

指针p的加减法运算

  • 指针p + N
    • p里面存储的地址值 + N * 指针所指向类型的占用字节数
  • 指针p - N
    • p里面存储的地址值 - N * 指针所指向类型的占用字节数

数组名

  • 存储的是数组首元素的地址
  • 等价于:一个指向数组首元素的指针
  • 数组名 + 1 的跨度：数组首元素的占用字节数
