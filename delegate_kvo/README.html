代理

  • 代理设计模式的作用:

    • 1.A对象监听B对象的一些行为，A成为B的代理
    • 2.一个对象B状态发生了改变想通知另一个对象A，A成为B的代理

  • 代理设计模式的总结：

    • 如果你想监听别人的一些行为，那么你就要成为别人的代理
    • 如果你想告诉别人一些事情，那么就让别人成为你的代理

  • 代理设计模式的开发步骤

    • 1.拟一份协议（协议名字的格式：控件名 + Delegate），在协议里面声明一些代理方法（一般代理方法都是@optional）
    • 2.声明一个代理属性：@property (nonatomic, weak) id<代理协议> delegate;
    • 3.在内部发生某些行为时，调用代理对应的代理方法，通知代理内部发生什么事
    • 4.设置代理：xxx.delegate = yyy;
    • 5.yyy对象遵守协议，实现代理方法

KVC\KVO

  • KVC(Key Value Coding)常见作用：给模型属性赋值
  • KVO(Key Value Observing)常用作用：监听模型属性值的改变
  • 面试题：KVO 是同步的，还是异步的？ 答案：同步的
  • KVO特点及隐患
    • 特点：监听某一个具体的属性！
    • "隐患"：一旦键值是在其他线程被修改，监听方法，同样会在其他线程被执行！ 如果存在多个线程，同时修改一个对象的属性，会出现“资源抢夺”的问题！
    • 无论监听多少个对象的键值，只要发生变化，都会调用同一个观察者方法！ 如果真的监听很多，会让观察者方法写起来非常复杂！
  • 优点：在实际开发中，有时候会遇到一些只有 KVO 能够胜任的开发工作！
  • 注意
  • 不要忘记删除观察者！

  • 可以通过 context 加以区分具体变化是由哪一个观察者事件产生的

  • KVO的使用步骤

// cc监听了aa的name属性的改变
[aa addObserver:cc forKeyPath:@"name" options: NSKeyValueObservingOptionOld context:nil];

// cc得实现监听方法
/**
 * 当监听到object的keyPath属性发生了改变
 */
- (void)observeValueForKeyPath:(NSString *)keyPath ofObject:(id)object change:(NSDictionary *)change context:(void *)context
{
    NSLog(@"监听到%@对象的%@属性发生了改变， %@", object, keyPath, change);
}

如果字典中的key跟模型中的属性名称不同重写setValue forKey方法

- (void)setValue:(id)value forKey:(NSString *)key
{
    if ([key isEqualToString:@"id"])
    {
        [self setValue:value forKeyPath:@"ID"];
    }
    else
    {
        [super setValue:value forKey:key];
    }
}

代理\通知\KVO\block的区别

  • 代理：1个对象只能告诉另1个对象发生了什么事
  • 通知：1个对象可以告诉N个对象发生了什么事
  • 通知不需要知道接收者是谁
  • 代理&block可以是双向的(可以调用可以有返回值)
  • 通知&KVO都是单向
  • 回调方法少用block
  • 回调方法多用代理
  • KVO和通知都是观察者模式，不同的是，KVO是被观察者直接发送消息给观察者，是对象间的交互，而通知则是观察者和被观察者通过通知中心对象进行交互，即消息由被观察者发送到通知中心对象，再由中心对象发给观察者，两者之间并不进行直接的交互 KVO只能监听某一个具体的属性, 观察的属性必须使用string来定义。而通知监听不限于属性的变化，可以对多种多样的状态变化进行监听，监听范围广，使用灵活
