memory

  • __bridge NSString *
    • 将CoreFoundation框架的对象所有权交给Foundation框架来使用,但是Foundation框架中的对象并不能管理该对象内存
  • __bridge_transfer NSString *
    • 将CoreFoundation框架的对象所有权交给Foundation来管理,如果Foundation中对象销毁,那么我们之前的对象(CoreFoundation)会一起销毁
    • __bridge_retained:类型转换后，将相关对象的引用计数加1，原来的Core Foundation对象在不用时，需要调用CFRelease方法

将代码转换为ARC的时候要注意以下两点： 1、不要再使用retain、release、autorelease，可以直接将这些代码删除。ARC在编译时会自动在合适位置插入内存管理代码。 2、如果dealloc方法只是用于释放实例变量，那么dealloc方法也没有存在的必要了，可以直接删掉。在任何情况下都不需要使用release,ARC会自动处理好这些事情。如果仍然需要使用dealloc来做一些其他的事情（比如移除KVO观察）记得不要再调用[super dealloc] ，否则编译器会给出错误信息。 支持 arc -fobjc-arc 不支持arc -fno-objc-arc

可使用class和superclass方法来找到一个对象的类和超类。 用方法respondsToSelector检查对象是否响应一条消息 如果两个类需要知道对方，相互包含则代码无法编译，必须至少在两个头文件的一个之中使用一条@class指令。

Alloc做的几件事情 Alloc为新的对象分配内存，并且返回指向该内存的一个指针，alloc填充对象的isa实例变量，编译器自动为每个对象实例添加一个名为isa的实例变量。Isa指向关于对象的类的信息的一个结构。Alloc清空其他对象的内存，除了isa以外的所有实例变量都以一个0值开始。Alloc将新创建的对象引用计数设置为1

Init方法的返回类型为什么是id而不是指向特定类实例的指针 相同的init方法可以用于不同的类的接收者，这允许继承了声明该方法的初始类的那些类，继承并使用该方法。

在对象创建时，实例变量是由alloc初始化为0的

在为一个类设计初始化程序时，有两条规则必须遵守： 所有其它的类初始化程序，最终必须调用该类的而不是超类的指定初始化程序。 类的指定初始化程序必须调用其超类的指定初始化方法。

类本身是指针类型，在声明class变量时，没有必要使用*

和实例方法一样，类方法由子类继承，子类可覆盖一个继承的类方法。

对于一个给定的类，可以有任意多个分类，但每个分类必须有一个不同的名称。也必须声明并实现一套不同的方法。不能使用一个分类方法来覆盖另外一个分类方法。如果在不同的分类中用相同的名称声明了方法，则可能不会导致编译器或连接错误，但是，所使用的该方法的版本，则取决于分类载入的顺序。

@property语句提供了声明访问器方法的一种快捷方式。 @synthesize语句编译器为你编写指定的访问器方法。

实例变量名称可以与特性名称不同，如果想要使用不同的名称，你就必须在@synthesize语句中通知编译器。

如果一个类采用了协议，或者如果它继承了采用该协议的一个类，这个类就遵守该协议。 类可以采用多个协议，在协议的每对尖括号之间用逗号隔开。

如果你创建了一个对象，你拥有它，通过向其他人所创建的一个对象发送一条retain消息，你就获得该对象的所有权。当你用完自己所拥有的一个对象时，必须通过向其发送一条release消息，来放弃对其的所有权。

一个对象保持对另一个对象的引用而不保留它，这称之为对该对象进行弱引用。一个对象将自己的实例变量中保存的一个对象保留，这叫强引用。

Objc需要使用alloc申请内存。不同的是C直接调用free函数来释放内存，而oc并不直接调用dealloc来释放，整个objc都使用对象引用，而且每个对象有一个引用计数器，当计数器为0时，系统调用dealloc来释放内存。

当对象添加到任何类型的集合中的时候，这都会使该对象的有效引用增加。 赋值操作并不会使对象的引用计数增加，只是将对象的内存地址传递了，这两个指针指的是同一个对象。。

字符串常量的内存分配方式和其他对象不同，这种方式没有引用计数机制。 可变字符串对象的值可能在程序执行过程中发生变化，但是字符串常量是无法改变的，因此可变字符串对象拥有了引用计数。

当一个对象并不在自动释放池中时，即使调用了自动释放池的drain方法，它的引用计数仍然保持为1
